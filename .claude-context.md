# Contexto del Proyecto Mini-GitHub

## Resumen del Proyecto
Proyecto académico para **Arquitectura en la Nube y Microservicios** - recreación simplificada de GitHub.

## Arquitectura de Microservicios

### 1. Auth Service (Puerto 3001)
- **Responsabilidad:** Autenticación, registro, JWT, OIDC/Keycloak
- **Base de datos:** PostgreSQL (auth_db)
- **Tecnologías:** Node.js, TypeScript, Express, Prisma, Keycloak
- **Eventos:** user.created, user.updated, user.deleted

### 2. Repo Service (Puerto 3002)
- **Responsabilidad:** CRUD repositorios, gestión archivos, branches, stars, forks
- **Base de datos:** PostgreSQL (repos_db) + MinIO/S3
- **Tecnologías:** Node.js, TypeScript, Express, Prisma
- **Eventos:** repo.created, repo.updated, repo.deleted, file.uploaded

### 3. Issue Service (Puerto 3003)
- **Responsabilidad:** CRUD issues, comentarios, labels, asignación
- **Base de datos:** PostgreSQL (issues_db)
- **Tecnologías:** Node.js, TypeScript, Express, Prisma
- **Eventos:** issue.created, issue.updated, issue.closed, comment.created

### 4. Search Service (Puerto 3004)
- **Responsabilidad:** Indexación, búsqueda full-text
- **Base de datos:** Elasticsearch
- **Tecnologías:** Node.js, TypeScript, Express, Elasticsearch
- **Consume:** repo.created, repo.updated, repo.deleted

### 5. API Gateway (Puerto 8080)
- **Responsabilidad:** Enrutamiento, autenticación, rate limiting, CORS
- **Tecnologías:** Kong/Nginx o Express con http-proxy-middleware

## Infraestructura Compartida

### Bases de Datos
- PostgreSQL (3 instancias: auth_db, repos_db, issues_db)
- Elasticsearch (índices de búsqueda)
- Redis (caché y sesiones)

### Mensajería
- RabbitMQ (eventos asíncronos entre servicios)

### Almacenamiento
- MinIO/S3 (archivos de repositorios, avatares)

### Autenticación
- Keycloak (OIDC/SSO)
- Federación: GitHub OAuth, Google OAuth, usuario/contraseña local
- JWT con expiración (Access token 15m, Refresh token 7d)

## Stack Tecnológico

### Backend (todos los servicios)
- Node.js 18+
- TypeScript
- Express.js o Fastify
- Prisma ORM (para PostgreSQL)
- Zod/Joi (validación)
- bcrypt (hashing contraseñas)

### Frontend
- React 18 + TypeScript
- Vite
- Tailwind CSS
- Zustand/Redux Toolkit (state)
- TanStack Query/Axios (HTTP)
- React Router v6

### DevOps
- Docker + Docker Compose (desarrollo)
- Kubernetes (producción)
- Terraform (IaC - opcional)

## Patrones de Arquitectura
- **Microservicios:** Servicios independientes y desplegables
- **Database per Service:** Cada servicio tiene su propia BD
- **API Gateway:** Punto único de entrada
- **Event-Driven:** Comunicación asíncrona vía RabbitMQ
- **CQRS:** Separación lecturas/escrituras (Search Service)

## Funcionalidades Principales

### Incluidas
- ✅ Autenticación (registro, login, SSO Keycloak)
- ✅ Repositorios (CRUD, público/privado, branches, fork)
- ✅ Archivos (upload, download, visualizar)
- ✅ Issues (CRUD, comentarios, labels)
- ✅ Pull Requests (crear, revisar, aprobar, merge)
- ✅ Búsqueda (repos y usuarios)
- ✅ Colaboración (stars, roles)

### Excluidas
- ❌ CI/CD pipelines (workflows, runners)
- ❌ Git protocol real (solo simulación)
- ❌ Notificaciones (email, push, WebSockets)
- ❌ Búsqueda de código
- ❌ Organizaciones avanzadas

## Modelo de Datos - Entidades Principales

### Auth Service
- `users` (id, username, email, password_hash, avatar_url, bio, created_at)
- `oauth_accounts` (id, user_id, provider, provider_account_id, access_token)
- `sessions` (id, user_id, token, expires_at)

### Repo Service
- `repositories` (id, name, description, is_private, owner_id, created_at)
- `repository_permissions` (id, user_id, repository_id, role)
- `branches` (id, name, repository_id, is_default)
- `commits` (id, message, author_id, repository_id, branch_id)
- `files` (id, path, content, commit_id)
- `stars` (id, user_id, repository_id)
- `pull_requests` (id, repository_id, source_branch_id, target_branch_id, author_id, title, status)

### Issue Service
- `issues` (id, repo_id, number, title, body, state, author_id, assignee_id)
- `labels` (id, repo_id, name, color, description)
- `issue_labels` (issue_id, label_id)
- `comments` (id, body, author_id, issue_id, pull_request_id)

## Roles y Permisos
- **Owner:** Acceso total al repositorio
- **Developer:** Subir archivos, crear issues, comentar
- **Reporter:** Solo lectura

## Endpoints API (Ejemplos)

### Auth Service
- POST `/api/auth/register` - Registro
- POST `/api/auth/login` - Login
- GET `/api/auth/login/keycloak` - SSO Keycloak
- GET `/api/auth/callback/keycloak` - Callback OIDC
- GET `/api/auth/me` - Usuario actual

### Repo Service
- GET/POST `/api/repos` - Listar/crear repos
- GET/PATCH/DELETE `/api/repos/:owner/:repo` - Gestión repo
- GET/PUT/DELETE `/api/repos/:owner/:repo/contents/*path` - Archivos
- PUT/DELETE `/api/repos/:owner/:repo/star` - Stars

### Issue Service
- GET/POST `/api/repos/:owner/:repo/issues` - Issues
- GET/PATCH `/api/repos/:owner/:repo/issues/:number` - Issue específico
- POST `/api/repos/:owner/:repo/issues/:number/comments` - Comentarios

### Search Service
- GET `/api/search/repositories?q=` - Buscar repos
- GET `/api/search/users?q=` - Buscar usuarios

## Comunicación entre Servicios

### Eventos RabbitMQ
- `user.created`, `user.updated`, `user.deleted` (Auth)
- `repo.created`, `repo.updated`, `repo.deleted`, `file.uploaded` (Repo)
- `issue.created`, `issue.updated`, `issue.closed`, `comment.created` (Issue)

### Consumidores
- Search Service indexa cuando recibe eventos de Repo Service
